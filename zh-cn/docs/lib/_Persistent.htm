#Persistent

让脚本持续运行(即直到用户关闭或遇到 ExitApp).

#Persistent

如果以下条件中的任何一个为真, 则脚本持续运行:

  • 至少有一个热键或热字串已经在脚本中定义, 或由 Hotkey 命令或 Hotstring 函数创建, 即使它没有被启用.
  • 键盘钩子或鼠标钩子已经被安装.
  • 脚本中包含任何对 Gui 的使用, 即使它没有被调用.
  • 脚本包含任何对 OnMessage() 的使用, 或者动态调用了它, 或者用 Func() 检索了一个引用.
  • Input 命令已经被调用.
  • 脚本中的任何地方存在 #Persistent 指令.

使用这个指令可以防止脚本在自动执行部分(脚本的顶部部分) 完成后退出. 这在脚本包含定时器和/或自定义菜单项但不符合上面列出的任何条件的情况下很有用.

如果添加此指令到现有的脚本中, 您可能需要把一些或全部的 Exit 改为 ExitApp. 这是由于 Exit 不会终止持续运行的脚本; 它只会终止当前线程.

[v1.0.16+]: 此指令同时也让脚本单实例运行. 要覆盖这种行为或改变单实例的方式, 请参阅 #SingleInstance.

和其他指令一样, #Persistent 不能有条件地执行.

相关

#SingleInstance, SetTimer, Menu, Exit, ExitApp

示例

使脚本永久运行; 也就是说, 如果可以的话, 它不会自动退出.
