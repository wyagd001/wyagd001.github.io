转义序列

转义字符 `(反单引号或重音符) 用来表明在紧跟其后的字符不能像平常那样进行解释. 该字符位于大多数英式键盘的左上角. 使用这个字符而不是反斜杠, 以避免在文件路径中需要两个反斜杠.

输入这些字符 得到这些字符
`` ` (原义的重音符; 也就是说, 两个连续的转义字符会产生一个单独的原义字符)
`;

;(原义的分号)

注意: 当分号的左边有任何字符时(除了 空格或制表符), 没有必要进行转义, 因为它不会被解释为注释.
`: :(原义的冒号). 这仅在热字串触发缩写中是必要的.
`{ {(键盘按键). 重映射按键到 { 时才有效, 并且是必需的.
`n 新行(换行/LF)
`r 回车(CR)
`b 退格
`t tab(制表符, 很典型的水平移位)
`s 空格
`v 垂直 tab -- 对应的 Ascii 值为 11. 在一些应用程序中可以通过键入 Control+K 让它显示出来.
`a 警告(铃) -- 对应的 Ascii 值为 7. 在一些应用程序中可以通过键入 Control+G 让它显示出来.
`f 进纸 -- 对应的 Ascii 值为 12. 在一些应用程序中可以通过键入 Control+L 让它显示出来.
Send 在默认(非文本(Raw)) 模式下, 使用 Send 函数或热字串时, 这些字符(如, {}^!+#) 具有特殊含义. 因此, 要使用它们的原义字符, 必须用大括号将它们括起来. 例如: Send "{^}{!}{{}".
`" 或 `' 单引号(') 和双引号(") 所拥有的功能相同, 只是单引号中包含的字符串可以包含原义双引号, 反之亦然. 因此, 要在原义字符串中包含实际的引号, 请转义引号或将字符串用相反类型的引号括起来. 例如: Var := "The color `"red`" was found." 或 Var := 'The color "red" was found.'.

示例

#1

MsgBox "Line 1`nLine 2"
