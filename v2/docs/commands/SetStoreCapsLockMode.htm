SetStoreCapsLockMode

在 Send 之后是否恢复 CapsLock 的状态.

SetStoreCapsLockMode Mode

参数

Mode

类型: 整数(布尔值)

下列值之一:

1 或 True: 这是所有脚本的初始设置, 如果 Send 在发送时需要临时更改 CapsLock 键的状态, 则在发送后会恢复 CapsLock 键为之前的状态.

0 或 False: CapsLock 键的状态从不会改变. 所以, 如果在 Send 发送时 CapsLock 恰巧是打开的, 则它所发送的字符是大小写颠倒的.

备注

这意味着在 Send 和 ControlSend 执行时, CapsLock 键的状态不总是关闭的. 即使 CapsLock 键被成功关闭了, 在发送按键后也可能不会恢复它原来的状态.

此函数极少使用, 因为在大多数情况中默认行为是最佳的.

该设置被盲从模式和文本模式忽略; 也就是说, 在这些情况下 CapsLock 的状态不会改变.

内置变量 A_StoreCapsLockMode 包含当前设置(1 或 0).

每个新运行的线程(如 热键, 自定义菜单项或定时子程序) 都会以此函数的默认设置开始. 这个默认设置可以通过在脚本启动中使用此函数来改变.

相关

SetCaps/Num/ScrollLockState, Send, ControlSend

示例

#1

SetStoreCapsLockMode False
