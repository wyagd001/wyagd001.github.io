#MaxThreadsPerHotkey

设置每个热键或热字串可同时存在的最大线程数.

#MaxThreadsPerHotkey Value

参数

Value

类型: 整数

给定热键/热字串子例程可以启动的最大线程数(限制 255).

备注

此设置用于控制允许同时存在多少给定热键或热字串子例程的 "实例". 例如, 如果一个热键的最大线程数为 1, 并且在其子例程已经运行时再次按下该键, 则该键将被忽略. 这有助于防止意外按下两次. 但是, 如果您希望缓冲这些按键而不是忽略这些按键 -- 可能要增加键盘自动重复功能的响应性 -- 请使用 #MaxThreadsBuffer.

与 #MaxThreads 不同, 此设置 不是 全局的. 而是将其放置在您希望受到其影响的第一个热键标签之前, 这将导致所有后续的热键都使用该值, 直到遇到该指令的另一个实例为止.

无论此设置如何, 其第一行是 ExitApp, Pause, Edit, Reload, KeyHistory, ListLines, ListVars 或 ListHotkeys 的任何热键子例程都将始终运行.

如果 #MaxThreads 设置的值低于此设置, 则其优先.

如果脚本中未指定此指令, 则其行为就像设置为 1 一样.

相关

示例

#MaxThreadsPerHotkey 3
